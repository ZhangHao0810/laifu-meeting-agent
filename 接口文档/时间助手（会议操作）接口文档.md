## 1. 新增单个会议接口

**接口用途**  
用于创建/新增一个会议预约。成功后返回 `meetingId`，可用于后续操作（如添加与会人、编辑、取消会议）。

**请求方法**：POST  
**请求地址**：  
`https://im.zhongfu.net/gateway/cloudwork/meeting/create?accessToken={token}`

**Content-Type**：`application/json`

### 请求参数（Body）

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| openId | string | 是 | 会议发起人/创建人的 openId (IM_OPEN_ID) | "64db4447e4b00d5d7f029ee2" |
| title | string | 是 | 会议标题 | "测试会议" |
| content | string | 否 | 会议内容/描述 | "新增会议测试" |
| roomId | string | 是 | 会议室ID（从 freeRooms 接口获取） | "64dde8fdf2e340001db9acc" |
| startDate | long | 是 | 会议开始时间戳（毫秒） | 17702460000000 |
| endDate | long | 是 | 会议结束时间戳（毫秒） | 17702964000000 |
| noticeTimes | array(number) | 否 | 提醒时间 | [5, 15, 60] |
| actors | array(string) | 否 | 协作人/与会人 openId 集合 | ["xxx", "xxx"] |
| type | string | 否 | 会议类型 | null |

> [!TIP]
> **参数说明**：
> - `noticeTimes`：提醒时间（-1=不提醒、0=开始时间提醒、15=开始前15分钟、60=开始前1小时）
> - `type`：会议类型（null=普通会议、sign=线下签到类会议、voice=语音类会议）

> [!IMPORTANT]
> **注意事项**：
> - 时间戳必须为毫秒级
> - `startDate` < `endDate`，且需在会议室可用时间内
> - 如需邀请与会人，可通过 `actors` 参数添加

**请求示例**

```json
{
  "openId": "64db4447e4b00d5d7f029ee2",
  "title": "测试会议",
  "content": "新增会议测试",
  "roomId": "64dde8fdf2e340001db9acc",
  "startDate": 17702460000000,
  "endDate": 17702964000000,
  "noticeTimes": [5, 15, 60],
  "actors": ["644b417ae4b00d5d7f028c43", "644b4447e4b00d5d7f029ee2"]
}
```

### 响应示例（成功）

```json
{
  "data": {
    "meetingId": "698450b0b5f7a4aa49152f1"
  },
  "error": "success",
  "errorCode": 200,
  "success": true
}
```

### 响应字段说明

#### 核心字段

| 字段 | 含义 | 示例值 | 备注 |
|------|------|--------|------|
| **meetingId** | **会议唯一ID** | "698450b0b5f7a4aa49152f1" | ⭐ 新创建的会议ID，用于后续操作 |

#### 响应结构字段

| 字段 | 含义 | 说明 |
|------|------|------|
| data.meetingId | 会议ID | 新创建的会议唯一标识 |
| errorCode | 状态码 | 200=正常 |
| success | 是否成功 | true/false |
| error | 错误信息 | "success" |

### 常见状态与处理

- `success: true` + `meetingId` 存在 → 会议创建成功
- `errorCode: 200` → 请求成功
- 保存 `meetingId` → 用于后续编辑、取消、查询与会人等操作

## 2. 修改单个会议接口

**接口用途**  
修改已创建会议的标题、描述、时间、参会人等信息。支持增删参会人（`addActors` / `delActors`），无需全量覆盖。

**请求方法**：POST  
**请求地址**：  
`https://im.zhongfu.net/gateway/cloudwork/meeting/modify?accessToken={token}`

**Content-Type**：`application/json`

### 请求参数（Body）

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| id | string | 是 | 会议ID（从 create 接口返回的 meetingId） | "698456530b5f7a4aa491547b" |
| openId | string | 是 | 操作人 openId（必须有修改权限，通常为创建者） | "64db4447e4b00d5d7f029ee2" |
| title | string | 否 | 新的会议标题 | "测试会议22222222" |
| content | string | 否 | 新的会议内容/描述 | "更新会议测试" |
| startDate | long | 否 | 新的开始时间戳（毫秒） | 1770246800000 |
| endDate | long | 否 | 新的结束时间戳（毫秒） | 1770296400000 |
| roomId | string | 否 | 会议室ID | "xxxxx" |
| noticeTimes | array(number) | 否 | 提醒时间 | [5, 15, 60] |
| addActors | array(string) | 否 | 要新增的与会人 openId 数组 | ["64db4210e4b00d5d7f029815"] |
| delActors | array(string) | 否 | 要移除的与会人 openId 数组 | ["64db4218e4b00d5d7f029815"] |
| type | string | 否 | 会议类型 | null |

> [!TIP]
> **参数说明**：
> - `noticeTimes`：提醒时间（-1=不提醒、0=开始时间提醒、15=开始前15分钟、60=开始前1小时）
> - `type`：会议类型（null=普通会议、sign=线下签到类会议、voice=语音类会议）
> - `addActors` / `delActors`：支持增量修改参会人，无需全量覆盖

> [!IMPORTANT]
> **注意事项**：
> - 除 `id` 和 `openId` 外，所有字段均为可选，只传需要修改的部分
> - 时间调整后需确保新时间段会议室可用（可先查 freeRooms）
> - 修改成功后 `data` 为 `null`，`success` 为 `true`

**请求示例**

```json
{
  "id": "698456530b5f7a4aa491547b",
  "openId": "64db4447e4b00d5d7f029ee2",
  "title": "测试会议22222222",
  "content": "更新会议测试",
  "startDate": 1770246800000,
  "endDate": 1770296400000,
  "addActors": ["64db4210e4b00d5d7f029815"],
  "delActors": ["64db4218e4b00d5d7f029815"]
}
```

### 响应示例（成功）

```json
{
  "data": null,
  "error": "success",
  "errorCode": 200,
  "success": true
}
```

### 响应字段说明

#### 响应结构字段

| 字段 | 含义 | 说明 |
|------|------|------|
| data | 返回数据 | 修改成功时为 null |
| errorCode | 状态码 | 200=正常 |
| success | 是否成功 | true/false |
| error | 错误信息 | "success" |

### 常见状态与处理

- `success: true` + `data: null` → 会议修改成功
- `errorCode: 200` → 请求成功
- 修改失败 → 检查是否有修改权限，或时间段是否冲突

## 4. 取消单个会议接口

**接口用途**  
取消/删除已创建的单个会议预约。成功后 `data` 为 `null`，`success` 为 `true` 表示操作完成。

**请求方法**：POST  
**请求地址**：  
`https://im.zhongfu.net/gateway/cloudwork/meeting/cancel?accessToken={token}`

**Content-Type**：`application/json`

> [!IMPORTANT]
> **权限要求**：只有会议创建者或有取消权限的用户才能操作。

### 请求参数（Body）

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| id | string | 是 | 会议ID（从 create 接口返回的 meetingId） | "698450b0b5f7a4aa49152f1" |
| openId | string | 是 | 操作人/会议发起人的 openId | "64db4447e4b00d5d7f029ee2" |

> [!CAUTION]
> **注意事项**：
> - 取消后，该会议在列表中消失（或标记为已取消，视系统实现）
> - 建议在取消前提示用户确认，避免误操作

**请求示例**

```json
{
  "id": "698450b0b5f7a4aa49152f1",
  "openId": "64db4447e4b00d5d7f029ee2"
}
```

### 响应示例（成功）

```json
{
  "data": null,
  "error": null,
  "errorCode": 0,
  "success": true
}
```

### 响应字段说明

#### 响应结构字段

| 字段 | 含义 | 说明 |
|------|------|------|
| data | 返回数据 | 取消成功时为 null |
| errorCode | 状态码 | 0=正常 |
| success | 是否成功 | true/false |
| error | 错误信息 | 成功时为 null |

### 常见状态与处理

- `success: true` + `data: null` → 会议取消成功
- `errorCode: 0` → 请求成功
- 取消失败 → 检查是否有取消权限，或会议是否已被取消

## 5. 按天查询工作圈下会议

**接口用途**  
按指定日期（一天）查询当前用户工作空间/可见的所有会议列表。常用于日历视图、日程列表、首页当天会议展示。

**请求方法**：POST  
**请求地址**：  
`https://im.zhongfu.net/gateway/cloudwork/meeting/queryByDay?accessToken={token}`

**Content-Type**：`application/json`

### 请求参数（Body）

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| day | long | 是 | 待查询的日期时间戳（当天任意时间戳都可以，毫秒级） | 17702577533345 |

> [!TIP]
> **参数说明**：`day` 传当天任意毫秒时间戳即可（建议用当天 00:00:00 的时间戳），系统会自动取当天范围。

**请求示例**

```json
{
  "day": 17702577533345
}
```

### 响应示例（成功）

```json
{
  "data": [
    {
      "id": "698456530b5f7a4aa491547b",

      "readStatus": 0,

      "cancelReason": "",
```

### 响应字段说明

#### 核心字段（data 数组中的会议对象）

| 字段 | 含义 | 示例值 | 备注 |
|------|------|--------|------|
| **id** | **会议ID** | "698456530b5f7a4aa491547b" | ⭐ 会议唯一标识 |

#### 会议室信息

| 字段 | 含义 | 示例值 | 备注 |
|------|------|--------|------|
| roomId | 会议室ID | "644df1d4f2e340001dbde8" | |
| roomOrderId | 会议室订单ID | "6984565342f8ce0001794bc" | |
| meetingPlace | 会议室地址 | null | 可能为空 |

| 字段 | 含义 | 示例值 | 备注 |
|------|------|--------|------|

| readStatus | 阅读状态 | 0 | 0=未读, 1=已读 |
| record | 是否有会议记录 | false | true/false |

#### 取消信息

| 字段 | 含义 | 示例值 | 备注 |
|------|------|--------|------|
| cancelDate | 取消时间戳 | 1770281926982 | 若取消则有值 |
| cancelReason | 取消原因 | "" | 取消时的原因说明 |

#### 其他字段

| 字段 | 含义 | 示例值 | 备注 |
|------|------|--------|------|
| createDate | 创建时间戳 | 0 | |
| doneTime | 完成时间戳 | 0 | |
| noticeTime | 提醒时间 | [] | 数组 |
| actors | 协作人列表 | null | 可能为空 |
| submitExperience | 是否提交体验 | true | |

#### 响应结构字段

| 字段 | 含义 | 说明 |
|------|------|------|
| data | 会议列表 | 数组，包含当天所有会议 |
| errorCode | 状态码 | 0=正常 |
| success | 是否成功 | true/false |
| error | 错误信息 | 成功时为 null |

### 常见状态与处理

- `success: true` + `data` 为数组 → 查询成功
- `data` 为空数组 → 当天无会议
- `errorCode: 0` → 请求成功
- 返回数组按时间顺序或创建顺序排序（实际以接口返回为准）
- 只返回用户有权限查看的会议（创建的、参与的、可见的）
- `cancelDate` 不为0 → 会议已被取消，显示取消信息

## 6. 按时间范围查询工作圈下会议

**接口用途**  
按自定义时间范围（例如一周、一月或任意起止时间）查询当前用户工作空间可见的所有会议列表。适合日历跨天、周视图、月视图、搜索历史/未来会议等场景。

**请求方法**：POST  
**请求地址**：  
`https://im.zhongfu.net/gateway/cloudwork/meeting/queryByRange?accessToken={token}`

**Content-Type**：`application/json`

### 请求参数（Body）

| 字段 | 类型 | 必填 | 说明 | 示例值 |
|------|------|------|------|--------|
| start | long | 是 | 开始时间戳（毫秒） | 1770278964112 |
| end | long | 是 | 结束时间戳（毫秒） | 1770296400000 |

> [!IMPORTANT]
> **注意事项**：
